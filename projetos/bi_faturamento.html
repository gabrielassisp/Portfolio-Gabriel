Reflexões
O que este projeto me ensinou
★
Modelagem antes do DAX
A qualidade da modelagem (esquema estrela, granularidade correta, relacionamentos bem definidos) determina 80% da facilidade de escrever DAX depois. Modelos mal estruturados criam medidas complexas onde deveriam ser simples.
◈
Contexto de filtro é tudo
Entender quando usar CALCULATE, ALLEXCEPT, KEEPFILTERS e USERELATIONSHIP foi o principal desafio. Medidas como a análise de cohort só são possíveis com domínio profundo de como o DAX propaga contextos.
▲
HTML dentro do DAX
Usar medidas que retornam HTML para criar KPI cards estilizados foi uma descoberta que elevou muito o visual do dashboard, sem depender de visuais de terceiros ou licenças extras.
Processo
Desafios e decisões técnicas
Desafios enfrentados
  • Dataset real com inconsistências: pedidos sem entrega, datas nulas e itens órfãos
  • Relacionamentos Many-to-Many entre fatos exigindo tratamento cuidadoso
  • Análise de cohort com lógica temporal complexa em DAX puro
  • Performance com 100k+ linhas e múltiplas medidas iterativas
Decisões tomadas
  • Todas as transformações feitas no Power Query, modelo apenas lê dados limpos
  • Colunas calculadas para atributos estáticos, medidas para tudo dinâmico
  • 69 medidas organizadas em 5 pastas temáticas para manutenção
  • Tabela $Medidas dedicada para centralizar toda a camada de negócio
