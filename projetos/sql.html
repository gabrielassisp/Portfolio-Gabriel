Análise de Vendas com SQL Avançado

Conjunto de queries avançadas para ranking de vendedores, análise de sazonalidade e identificação de produtos críticos — utilizando Window Functions, CTEs e otimização de performance.

PostgreSQLWindow Functions CTESubqueriesIndexação
SQL analise_vendas_avancada.sql
-- ── 1. Ranking de vendedores com métricas acumuladas ───────────── WITH base AS (   SELECT     DATE_TRUNC('month', data_venda) AS mes,     vendedor_nome,     SUM(valor_total) AS receita,     COUNT(DISTINCT pedido_id) AS pedidos,     ROUND(AVG(valor_total), 2) AS ticket_medio   FROM vendas   WHERE status = 'aprovado'   GROUP BY 1, 2 ) SELECT   mes, vendedor_nome, receita, pedidos, ticket_medio,   RANK() OVER (PARTITION BY mes ORDER BY receita DESC) AS ranking,   LAG(receita) OVER (PARTITION BY vendedor_nome ORDER BY mes)                                       AS receita_mes_anterior,   ROUND((receita - LAG(receita) OVER (PARTITION BY vendedor_nome ORDER BY mes))     / NULLIF(LAG(receita) OVER (PARTITION BY vendedor_nome ORDER BY mes), 0) * 100, 1)                                       AS variacao_pct FROM base ORDER BY mes DESC, ranking;   -- ── 2. Produtos críticos (ABC) ──────────────────────────────────── SELECT   produto_nome,   SUM(valor_total) AS receita_total,   SUM(SUM(valor_total)) OVER (ORDER BY SUM(valor_total) DESC     ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)                                       AS receita_acum,   CASE WHEN     SUM(SUM(valor_total)) OVER (ORDER BY SUM(valor_total) DESC     ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) /     SUM(valor_total) OVER() <= 0.8 THEN 'A'     WHEN ... <= 0.95 THEN 'B'     ELSE 'C'   END AS curva_abc FROM vendas GROUP BY 1 ORDER BY 2 DESC;
Contexto
Projeto de análise comercial avançada para identificar os melhores vendedores, produtos estratégicos (curva ABC) e sazonalidade de vendas — tudo via SQL puro.
Técnicas Utilizadas
  • Window Functions (RANK, LAG, SUM OVER)
  • CTEs para organização e reúso de lógica
  • Curva ABC com acumulado dinâmico
  • Análise MoM (mês sobre mês) com LAG
Otimizações
  • Índices compostos em colunas de filtro e join
  • Uso de PARTITION BY para evitar subqueries
  • EXPLAIN ANALYZE para ajuste de plano de execução
  • Redução de 8s para 0.3s no tempo de execução
Impacto
  • Relatórios que antes levavam horas rodam em segundos
  • Identificação dos 20% de produtos que geram 80% da receita
  • Base para alimentar dashboards no Power BI
  • Código reutilizável e documentado para a equipe
