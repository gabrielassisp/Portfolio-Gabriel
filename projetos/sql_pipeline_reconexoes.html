Identificação de Reconexões via Text Mining em SQL

Query de produção que identifica 3 tipos distintos de reconexão de clientes através de análise de texto no histórico de eventos usando regex nativo do PostgreSQL — com window functions, subqueries correlacionadas e CTEs encadeadas com dependência entre si.

PostgreSQLRegex ~*Window Function LAGSubquery correlacionadaProdução
-- ═══════════════════════════════════════════════════════════ -- CTEs: reconexao_tum | reconexao_tdois | reconexao_ttres -- Identifica 3 tipos distintos de reconexão via análise -- de histórico textual com regex (PostgreSQL ~*) -- ═══════════════════════════════════════════════════════════ -- TIPO 1: Reativação direta (Inativo → Ativo + novo serviço no mesmo dia) reconexao_tum AS ( SELECT cliente_servico.id_cliente_servico, ch_novo.data_cadastro::date AS data_habilitacao, uv.name AS vendedor, cliente.nome_razaosocial, servico.descricao AS servico, cidade.nome AS cidade FROM cliente_servico -- Evento 1: "Novo serviço de número (X) adicionado" — ancora pelo numero_plano JOIN cliente_historico ch_novo ON ch_novo.id_cliente = cliente_servico.id_cliente AND substring(ch_novo.historico FROM '(?i)servi[çc]o de n[úu]mero \((\d+)\)')::int = cliente_servico.numero_plano AND ch_novo.historico ~* 'Novo servi[çc]o de n[úu]mero \(\d+\) adicionado' -- Evento 2: "Inativo para Ativo" deve ser IMEDIATAMENTE anterior ao evento 1 JOIN cliente_historico ch_inativo ON ch_inativo.id_cliente = cliente_servico.id_cliente AND ch_inativo.historico ~* 'Dados cadastrais alterados.*O status do cadastro alterou ' || 'de Inativo para Ativo.*Modificação feita automaticamente' AND ch_inativo.data_cadastro::date = ch_novo.data_cadastro::date AND ch_inativo.id_cliente_historico = ( -- Garante sequência imediata no histórico SELECT max(ch2.id_cliente_historico) FROM cliente_historico ch2 WHERE ch2.id_cliente = ch_novo.id_cliente AND ch2.id_cliente_historico < ch_novo.id_cliente_historico ) LEFT JOIN users uv ON uv.id = ch_novo.id_usuario LEFT JOIN cliente ON cliente_servico.id_cliente = cliente.id_cliente LEFT JOIN servico ON cliente_servico.id_servico = servico.id_servico LEFT JOIN servico_status ON cliente_servico.id_servico_status = servico_status.id_servico_status LEFT JOIN cliente_servico_endereco ON cliente_servico.id_cliente_servico = cliente_servico_endereco.id_cliente_servico LEFT JOIN endereco_numero ON cliente_servico_endereco.id_endereco_numero = endereco_numero.id_endereco_numero LEFT JOIN cidade ON endereco_numero.id_cidade = cidade.id_cidade WHERE ch_novo.data_cadastro::date BETWEEN '2025-01-01' AND current_date AND (cliente_servico.id_servico_status = 11 OR (cliente_servico.id_servico_status = 9 AND cliente_servico.id_motivo_cancelamento = 3)) AND cliente.codigo_cliente NOT IN (12412,10384,11370) AND ch_novo.id_usuario NOT IN (1,79,81,82,88,92,135,142,371) -- Exclui se já consta em vendas recentes (≤ 7 dias) AND cliente_servico.id_cliente_servico NOT IN ( SELECT id_cliente_servico FROM vendas WHERE ch_novo.data_cadastro::date - vendas.data_habilitacao::date <= 7 ) -- Exclui troca de titularidade detectada via LAG no histórico AND cliente_servico.id_cliente NOT IN ( SELECT ht.id_cliente FROM ( SELECT ch2.id_cliente, ch2.historico, lag(ch2.historico) OVER ( PARTITION BY ch2.id_cliente ORDER BY ch2.data_cadastro, ch2.id_cliente_historico ) AS historico_anterior FROM cliente_historico ch2 ) ht WHERE ht.historico ~* 'status do cadastro alterou de Inativo para Ativo' AND ht.historico_anterior ~* 'troca de titularidade' ) ), -- TIPO 2: Cancelamento revertido + migração de plano (janela de 1 dia + 7 dias) reconexao_tdois AS ( SELECT cliente_servico.id_cliente_servico, ch_mig.data_cadastro::date AS data_habilitacao, u.name AS vendedor, cliente.nome_razaosocial, servico.descricao AS servico, cidade.nome AS cidade FROM cliente_servico JOIN cliente_servico csantigo ON cliente_servico.id_cliente_servico_antigo = csantigo.id_cliente_servico -- Passo 1: localiza o evento de reversão pelo número do plano antigo JOIN cliente_historico ch_rev ON ch_rev.id_cliente = cliente_servico.id_cliente AND substring(ch_rev.historico FROM '\((\d+)\)')::int = csantigo.numero_plano AND ch_rev.historico ~* 'O cancelamento do servi[çc]o.*foi revertido\.' -- Passo 2: localiza evento de atualização do plano novo (janela: mesmo dia + 1) JOIN cliente_historico ch_atual ON ch_atual.id_cliente = cliente_servico.id_cliente AND substring(ch_atual.historico FROM '\((\d+)\)')::int = cliente_servico.numero_plano AND ch_atual.historico ~* 'Dados do Servi[çc]o' AND ch_atual.data_cadastro > ch_rev.data_cadastro AND ch_atual.data_cadastro::date - ch_rev.data_cadastro::date <= 1 -- Passo 3: localiza linha de migração plano antigo → plano novo (janela: 7 dias) JOIN cliente_historico ch_mig ON ch_mig.id_cliente = cliente_servico.id_cliente AND ch_mig.historico ~* 'migrou\s+para' AND substring(ch_mig.historico FROM 'servi[cç]o\s*\((\d+)\)')::int = csantigo.numero_plano AND substring(ch_mig.historico FROM 'migrou\s+para\s+o\s+servi[cç]o\s*\((\d+)\)')::int = cliente_servico.numero_plano AND ch_mig.data_cadastro > ch_rev.data_cadastro AND ch_mig.data_cadastro <= ch_rev.data_cadastro + INTERVAL '7 day' LEFT JOIN users u ON u.name = coalesce(substring(ch_atual.historico FROM 'Vendedor:\s*([^;]+)'), u.name) LEFT JOIN cliente ON cliente_servico.id_cliente = cliente.id_cliente LEFT JOIN servico ON cliente_servico.id_servico = servico.id_servico LEFT JOIN cliente_servico_endereco ON cliente_servico.id_cliente_servico = cliente_servico_endereco.id_cliente_servico LEFT JOIN endereco_numero ON cliente_servico_endereco.id_endereco_numero = endereco_numero.id_endereco_numero LEFT JOIN cidade ON endereco_numero.id_cidade = cidade.id_cidade WHERE ch_atual.data_cadastro::date BETWEEN '2025-01-01' AND current_date AND (cliente_servico.id_servico_status IN (11) OR (cliente_servico.id_servico_status IN (9) AND cliente_servico.id_motivo_cancelamento IN (3))) AND ch_atual.id_usuario NOT IN (1,79,81,82,88,92,135,142,371) AND cliente_servico.id_cliente_servico NOT IN ( SELECT id_cliente_servico FROM vendas WHERE ch_atual.data_cadastro::date - vendas.data_habilitacao::date <= 7) AND cliente_servico.id_cliente_servico NOT IN (SELECT id_cliente_servico FROM reconexao_tum) ), -- TIPO 3: Renovação manual após reversão de cancelamento reconexao_ttres AS ( SELECT cliente_servico.id_cliente_servico, ch_atual.data_cadastro::date AS data_habilitacao, u.name AS vendedor, cliente.nome_razaosocial, servico.descricao AS servico, cidade.nome AS cidade FROM cliente_servico JOIN cliente_historico ch_rev ON cliente_servico.id_cliente = ch_rev.id_cliente AND substring(ch_rev.historico FROM '\((\d+)\)')::int = cliente_servico.numero_plano AND ch_rev.historico ~* 'O cancelamento do servi[çc]o.*foi revertido\.' LEFT JOIN cliente_historico ch_atual ON cliente_servico.id_cliente = ch_atual.id_cliente AND substring(ch_atual.historico FROM '\((\d+)\)')::int = cliente_servico.numero_plano AND ch_atual.historico ~* 'Servi[çc]o.*foi renovado manualmente\.' AND ch_atual.data_cadastro > ch_rev.data_cadastro AND ch_atual.data_cadastro::date - ch_rev.data_cadastro::date <= 1 LEFT JOIN users u ON u.id = coalesce(ch_atual.id_usuario, ch_rev.id_usuario) LEFT JOIN cliente ON cliente_servico.id_cliente = cliente.id_cliente LEFT JOIN servico ON cliente_servico.id_servico = servico.id_servico LEFT JOIN cliente_servico_endereco ON cliente_servico.id_cliente_servico = cliente_servico_endereco.id_cliente_servico LEFT JOIN endereco_numero ON cliente_servico_endereco.id_endereco_numero = endereco_numero.id_endereco_numero LEFT JOIN cidade ON endereco_numero.id_cidade = cidade.id_cidade WHERE ch_atual.data_cadastro::date BETWEEN '2025-01-01' AND current_date AND (cliente_servico.id_servico_status IN (11) OR (cliente_servico.id_servico_status IN (9) AND cliente_servico.id_motivo_cancelamento IN (3))) AND coalesce(ch_atual.id_usuario, ch_rev.id_usuario) NOT IN (1,79,81,82,88,92,135,142,371) AND cliente_servico.id_cliente_servico NOT IN ( SELECT id_cliente_servico FROM vendas WHERE ch_atual.data_cadastro::date - vendas.data_habilitacao::date <= 7) AND cliente_servico.id_cliente_servico NOT IN (SELECT id_cliente_servico FROM reconexao_tum) AND cliente_servico.id_cliente_servico NOT IN ( SELECT id_cliente_servico FROM reconexao_tdois WHERE ch_atual.data_cadastro::date - reconexao_tdois.data_habilitacao <= 30) ) -- SELECT FINAL — consolida todos os tipos numa única visão SELECT * FROM vendas UNION ALL SELECT * FROM reconexao_tum UNION ALL SELECT * FROM reconexao_tdois UNION ALL SELECT * FROM reconexao_ttres ORDER BY data_habilitacao DESC;
Contexto
Parte mais complexa do pipeline comercial. Identifica clientes que reconectaram ao serviço — o que conta como nova venda — distinguindo-os de reativações internas, trocas de titularidade e duplicatas já capturadas pela CTE de vendas.
Técnicas Utilizadas
  • Regex com ~* para parsing de texto livre no historico
  • Window Function LAG com PARTITION BY para garantir sequência de eventos
  • Subquery correlacionada para validar imediatamente anterior no histórico
  • 3 CTEs com exclusão cruzada (cada tipo exclui o que o anterior já capturou)
Os 3 tipos de reconexão
  • Tipo 1 (tum): Inativo→Ativo + novo serviço no mesmo dia
  • Tipo 2 (tdois): Reversão de cancelamento + migração de plano (janela 7 dias)
  • Tipo 3 (ttres): Reversão de cancelamento + renovação manual (janela 1 dia)
  • Exclusão de troca de titularidade via LAG no histórico de eventos
Impacto
  • Elimina contagem incorreta de reconexões como vendas novas
  • Atribui corretamente o vendedor responsável via parsing de texto
  • Garante que cada reconexão seja contada apenas uma vez (deduplicação)
  • Consolidado com o pipeline de vendas via UNION ALL no SELECT final
